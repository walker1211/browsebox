# Security Policy

## Reporting vulnerabilities

Use GitHub private vulnerability reporting if it is available for this repository.

If private reporting is not available, open a minimal public issue asking for private follow-up. Do not include secrets, private configuration, subscription URLs, proxy credentials, local runtime files, logs with tokens, or exploit details in a public issue.

## Scope

Security reports are welcome for browsebox behavior that could affect local process safety, temporary mihomo isolation, controller exposure, proxy credential handling, runtime file permissions, or accidental mutation of the user's main Clash Verge/mihomo selector.

## Handling expectations

Please include the affected version or commit, operating system, a concise impact summary, and minimal reproduction steps that do not disclose sensitive data.
